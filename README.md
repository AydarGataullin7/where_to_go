# Куда пойти — Москва глазами Артёма

Интерактивная карта интересных мест Москвы. Проект позволяет просматривать места на карте, знакомиться с их описаниями и фотографиями. Административная панель предоставляет удобные инструменты для управления контентом.

## Демо

Проект доступен по адресу: [ссылка будет добавлена после деплоя]

## Возможности

- Интерактивная карта с метками мест
- Просмотр подробной информации о месте (описание, фотографии, галерея)
- Административная панель для управления местами и фотографиями
- WYSIWYG-редактор для форматирования описаний
- Drag-and-drop сортировка фотографий в админке
- JSON API для получения данных о местах

## Технологии

- Python 3.11
- Django 5.2
- SQLite (разработка) / PostgreSQL (продакшн)
- Leaflet (карты)
- TinyMCE (визуальный редактор)
- django-admin-sortable2 (сортировка)
- environs (переменные окружения)

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/AydarGataullin7/where_to_go.git
cd where_to_go
```

### 2. Создайте и активируйте виртуальное окружение

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Настройте переменные окружения

Создайте файл .env в корне проекта со следующим содержимым:

```env
SECRET_KEY=ваш-секретный-ключ
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
```

Описание переменных:

- `SECRET_KEY` - Секретный ключ Django. Используется для криптографических подписей (сессии, cookies, CSRF-защита). Обязательная. Должен быть уникальной и непредсказуемой строкой.
- `DEBUG` - Режим отладки. При True Django показывает подробные сообщения об ошибках (трассировка стека, локальные переменные, настройки). Никогда не включайте `DEBUG=True` на боевом сервере.
- `ALLOWED_HOSTS` - Список доменных имён, на которых может работать сайт. Защита от атак подмены `HTTP Host`. Для разработки достаточно `127.0.0.1`,`localhost`. Для продакшена укажите реальные домены через запятую.

⚠️ Важно: Для продакшена установите DEBUG=False и укажите реальные домены в `ALLOWED_HOSTS`. Файл .env добавлен в .gitignore и не попадёт в репозиторий.

### 5. Примените миграции

```bash
python manage.py migrate
```

### 6. Создайте суперпользователя

```bash
python manage.py createsuperuser
```

### 8. Запустите сервер разработки

```bash
python manage.py runserver
```

Откройте браузер и перейдите по адресу: http://127.0.0.1:8000

## Административная панель

После запуска сервера административная панель доступна по адресу: http://127.0.0.1:8000/admin

## Управление местами

- Добавление/редактирование/удаление мест
- Заполнение названия, координат, короткого и полного описания

- Превью фотографий прямо в админке

## Загрузка данных из JSON

Для наполнения сайта данными используйте management-команду `load_place`. Она скачивает JSON-файл по указанному URL и сохраняет место с фотографиями в базу данных.

### Формат JSON-файла

```json
{
    "title": "Название места",
    "description_short": "Короткое описание",
    "description_long": "<p>Полное описание с HTML-разметкой</p>",
    "coordinates": {
        "lng": 37.64912239999976,
        "lat": 55.77754550000014
    },
    "imgs": ["https://example.com/photo1.jpg", "https://example.com/photo2.jpg"]
}
```

### Пример запуска

```bash
python manage.py load_place https://raw.githubusercontent.com/devmanorg/where-to-go-places/master/places/moscow_legends.json
```

### Пример вывода

```text
Loaded: Экскурсионная компания «Легенды Москвы»
```

### Загрузка нескольких мест

Вы можете загрузить несколько мест, выполнив команду для каждого JSON-файла:

```bash
python manage.py load_place https://example.com/place1.json
python manage.py load_place https://example.com/place2.json
python manage.py load_place https://example.com/place3.json
```
